# Test Procedures

This folder contains test procedures for CSF subcategory assessments.

## What Goes Here

Step-by-step procedures an auditor would follow to assess a specific CSF subcategory. Each file should include:

- **CSF Subcategory** - The specific control being tested (e.g., DE.AE-02)
- **Test Steps** - Numbered procedures for conducting the assessment
- **Evidence Requests** - Documents/artifacts needed from the organization
- **Related Artifacts** - Links to supporting documentation

## File Naming Convention

```
{CSF-ID}-{Short-Description}.md

Examples:
- DE-AE-02-Adverse-Event-Analysis.md
- GV-SC-04-Supplier-Criticality.md
- PR-AT-01-Security-Awareness-Training.md
```

## Template

See any existing file in this folder for the standard format.

## Contributing

1. Pick an unassigned CSF subcategory from the [Issues](https://github.com/CPAtoCybersecurity/csf_profile/issues)
2. Create your test procedure following the naming convention
3. Submit a Pull Request referencing the issue number
